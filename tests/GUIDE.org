* Pair Cycle Test Template

Use this template when adding reducer tests around head and tail transitions.

** Scenario skeleton

#+begin_src python
def test_<scenario_name>() -> None:
    # Given: initial pair state
    state = PairCycleState(...)

    # And: one typed reducer move
    move = EggMove(
        kind=EggMoveKind.<KIND>,
        occurred_at=datetime.now(timezone.utc),
        symbol="PI_XBTUSD",
        reply={...},
    )

    # When: reducer processes the move
    next_state, commands = step_pair(state, move)

    # Then: assert state transition
    assert next_state.head_state == HeadState.<EXPECTED>
    assert next_state.tail_mode == TailState.<EXPECTED>
    assert next_state.played_quantity == <EXPECTED_FLOAT>

    # And: assert command shape
    assert commands == (...)  # or len(commands) == N
#+end_src

** Behaviour checklist

- Terminal guard:
  `FAILED` and `CLOSED` states ignore later contradictory moves.
- Idempotency:
  replay of the same move does not emit duplicate `PLACE` tail command.
- Quantity safety:
  tail order quantity uses played quantity when present.
- Source-of-truth split:
  `HEAD_SUBMITTED` marks submitted only; private confirmation move hooks tail.
- Tail policy:
  `HEAD_FAILED` never hooks tail; `HEAD_CANCELED_ZERO_FILL` may hook tail by rule.

** Suggested test names

- `test_partial_fill_tail_uses_played_quantity_not_planned_quantity`
- `test_failed_head_ignores_later_played_event`
- `test_closed_head_ignores_later_partial_fill_event`
- `test_duplicate_head_filled_does_not_submit_second_tail`
- `test_rest_ack_marks_submitted_only`
- `test_private_confirmation_required_for_tail_hook`
