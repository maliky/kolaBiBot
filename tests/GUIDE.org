* Pair Cycle Test Template

Use this template when adding reducer tests around head and tail transitions.

** Scenario skeleton

#+begin_src python
def test_<scenario_name>() -> None:
    # Given: initial pair state
    state = PairCycleState(...)

    # And: one typed reducer move
    move = EggMove(
        kind=EggMoveKind.<KIND>,
        occurred_at=datetime.now(timezone.utc),
        symbol="PI_XBTUSD",
        reply={...},
    )

    # When: reducer processes the move
    next_state, intents = step_pair(state, move)

    # Then: assert state transition
    assert next_state.head_state == HeadState.<EXPECTED>
    assert next_state.tail_state == TailState.<EXPECTED>
    assert next_state.tail_mode == TailMode.<EXPECTED_OR_NONE>
    assert next_state.played_quantity == Decimal("<EXPECTED>")

    # And: assert intent shape
    assert intents == (...)  # or len(intents) == N
#+end_src

** Behaviour checklist

- Terminal guard:  `FAILED` and `CLOSED` states ignore later contradictory moves.
- Idempotency:  replay of the same move does not emit duplicate `PLACE` tail command.
- Quantity safety:  tail order quantity uses played quantity when present.
- Intent boundary:  reducer returns intents only; exchange payload building lives at interpreter boundary.
- Source-of-truth split:  `HEAD_SUBMITTED` marks submitted only; confirmed private events are classified into the canonical four moves.
- Tail policy:  `not_played_canceled` never hooks tail; only strategically played confirmed moves may hook or amend a tail.
