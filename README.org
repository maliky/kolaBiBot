Kolabi Bot

* Presentation
Kolabi is a typed, DB-grounded, async order-pair bot for derivatives/perpetual markets. The active production path is the TSV strategy runtime in =kolabi.bot=, backed by public market-data feeders and private account/order feeders in =kolabi.tree=. Kraken Futures is the primary live path. Binance USD-M Futures support is present in the shared adapter/feed layer and is being brought up behind the same DB-grounded architecture.

The core strategy unit is an order pair:
- a =head= order enters or opens the position;
- a =tail= order protects or closes the position;
- the pair can repeat, wait, update its tail, or activate another pair through a hook.

* Supported Platforms

- =kraken= is the current primary platform for demo and live Kraken Futures.
- =binance= has adapter and feed support for USD-M Futures, but should be treated as a bring-up path until its full operational runbook is validated.
- =bitmex= remains only as legacy compatibility and historical test context.

The bot invocation runs one platform at a time. TSV rows may carry platform-specific symbols directly; no canonical symbol mapping table is used.

* Strategy Conditions

Each pair can be gated by:
- an entry window, =tps_run=, in minutes from launch;
- a price interval, =prix=, interpreted by =atype=;
- a head timeout, =tOut=, for unfilled head orders;
- a repeat count, =essais= / =-n=;
- a pause between repetitions, =pause= / =-p=;
- a hook dependency, where a pair starts only after another pair reaches the required terminal condition.

Important runtime semantics:
- =tps_run= controls when a new pair attempt may start. Once an attempt starts, it is not stopped only because the entry window closes.
- =tOut= is stricter: if the head is not filled before the timeout, the head is cancelled and the attempt may repeat according to its repeat policy.
- Tail stops are grounded on private fill/order evidence, not on a guessed public price.
- Tail updates are no-widen by design: the pure reducer should not emit a tail amend that worsens the protected stop.

* Price, Quantity, And Tail Units

The compatibility vocabulary is encoded in =atype=:
- =pD= means the head price interval is differential from the reference price.
- =p%= means the head price interval is percent from the reference price.
- =pA= means the head price interval is absolute.
- =qA= means absolute quantity.
- =q%= means quantity as a percentage where supported by the strategy path.
- =tD= means tail distance is differential.
- =t%= means tail distance is percent.
- =tA= means tail price is absolute.
- =oD= / =o%= control optional head offsets where the order type supports them.

The =prix= field is a two-bound interval, not a single price. Use semi-open operator style in TSV files when appropriate, for example =- -5= or =5 +=, so the parser can translate the condition into a bounded internal interval.

* TSV Strategy Files

Strategy files live under =orders/=.

The active TSV grammar is:

#+begin_example
name  tps_run  essais  tOut  pause  side  oType  oDelta  tType  tDelta  atype  quantity  tp  prix  hook  symbol
#+end_example

The final =symbol= column is optional. When omitted, the CLI =--symbol= is used. When present, the row uses the platform-specific instrument directly, for example =PI_XBTUSD= or =PF_ADAUSD=. A multi-instrument TSV run requires one shared market DB URL and public feeders for every active symbol.

Useful examples:
- =orders/demo_ada.tsv= documents compact grammar examples.
- =orders/demo_kraken_xbt.tsv= covers Kraken XBT combinations.
- =orders/chained_xbt.tsv= exercises chained order flow.
- =orders/rep_xbt.tsv= exercises repetition.
- =orders/pi_xbtusd_sell_plus1_tail_0p5.tsv= is a one-row tail example.

* DB And Feed Services

The runtime expects fresh DB-backed evidence before active execution:
- public market snapshots from =kolabi.tree.kraken= or =kolabi.tree.binance=;
- private order/fill/account state from =kolabi.tree.account= or =kolabi.tree.binance_account=;
- a critical private lane for open orders and fills;
- optional REST audit and tail telemetry lanes.

PostgreSQL is the supported local runtime backend. Use =scripts/kolabidb postgres start= and the lane URLs from =.env.postgres= for public, private, critical, audit, and telemetry storage.

Start the default Kraken demo lanes:

#+begin_src bash
scripts/kolabidb postgres start
scripts/kolabidb public start --environment demo --pair PI_XBTUSD
scripts/kolabidb private start --environment demo --account-scope default
#+end_src

Check service status and logs:

#+begin_src bash
scripts/kolabidb public status --environment demo --pair PI_XBTUSD
scripts/kolabidb private status --environment demo --account-scope default
scripts/kolabidb public logs --environment demo --pair PI_XBTUSD
scripts/kolabidb private logs --environment demo --account-scope default
#+end_src

* Operator Commands



#+begin_src bash
  # List instruments and find exact symbols:  
  python -m kolabi.bargain.cli --exchange kraken --environment live instruments --contains ADA

  # Validate a selected symbol:
  python -m kolabi.bargain.cli --exchange kraken --environment demo --symbol PI_ADAUSD check-symbol

  # Read account state:
  python -m kolabi.bargain.cli --exchange kraken --environment demo --symbol PI_XBTUSD balance
  python -m kolabi.bargain.cli --exchange kraken --environment demo --symbol PI_XBTUSD position
  python -m kolabi.bargain.cli --exchange kraken --environment demo --symbol PI_XBTUSD open-orders
  python -m kolabi.bargain.cli --exchange kraken --environment demo --symbol PI_XBTUSD trigger-orders

  # Emergency cleanup:
  python -m kolabi.bargain.cli --exchange kraken --environment demo --symbol PI_XBTUSD close-all
#+end_src

* Running The Bot
#+begin_src bash
  # Dry-run a TSV strategy:
  python -m kolabi.bot run --strategy orders/demo_ada.tsv --exchange kraken --symbol PI_XBTUSD --environment demo --dry-run

  # Run a TSV strategy through the async runtime:
  python -m kolabi.bot run --strategy orders/rep_xbt.tsv --exchange kraken --symbol PI_XBTUSD --environment demo

  # Run one compatibility-vocabulary order pair:
  python -m kolabi.bot run-once --symbol PI_XBTUSD --environment demo -m XSellTail -t 0 1440 -O 60 -x 1 2 -q 1 -T 0.5 -o L -y S- -c sell -a qAt%pD --dry-run

  # Preflight runtime readiness:
  python -m kolabi.bot preflight --exchange kraken --symbol PI_XBTUSD --environment demo
#+end_src

Live/preprod should be launched only after a clean demo run, fresh public/private DB state, no unexpected open orders or positions, and an explicit dry-run of the exact strategy. Start live with small size, =--max-active-pairs 1=, =--rest-max-inflight 1=, and a non-zero =--rest-min-interval=.

* Installation

Use the repository Python environment declared by =.python-version=, then install the package and dependencies in editable mode:

#+begin_src bash
pyenv local kola
python -m pip install -r requirements.txt
python -m pip install -e .
#+end_src

Credential values should come from environment variables or a local env file that is not committed. Kraken defaults are:

#+begin_example
KRAKEN_FUTURE_DEMO_API_KEY
KRAKEN_FUTURE_DEMO_API_SECRET
KRAKEN_FUTURE_API_KEY
KRAKEN_FUTURE_API_SECRET
#+end_example

Use =--api-key-env= and =--api-secret-env= when selecting a non-default account.

* Tests

Start with focused active-path tests:

#+begin_src bash
PYENV_VERSION=kola pytest tests/bot tests/tree/test_kraken_tree.py -q
python -m ruff check kolabi tests
#+end_src

Legacy tests and archived paths exist for behavioural reference, but the active regression surface is under =tests/bot=, =tests/tree=, =tests/exchanges=, and =tests/core=.

* Disclaimer

This is an educational and experimental trading system. It can submit real orders when run against a live platform. Review the code, use small size, keep exchange-side risk limits in place, and do not run unattended until the live preprod path has proven itself under your operating conditions.

Read order for the core code:

- Start with =kolabi/tree/kraken.py= and =kolabi/tree/account.py= to see how the public and private Kraken DBs are fed.
- Read =kolabi/shared/exchanges/kraken_adapter.py= next to inspect real order placement, amend, cancel, and REST mapping.
- Read =kolabi/bot/__main__.py= and =kolabi/bot/service.py= to see how =run=, =preflight=, and =run-once= enter the runtime.
- Read =kolabi/bot/tsv/parser.py= if you want to map one TSV row to one normalized bot strategy.
- Read =kolabi/bot/= for the active strategy runtime and =archives/runtime/= only for historical reference.
