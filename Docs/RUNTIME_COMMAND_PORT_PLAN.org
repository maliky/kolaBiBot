#+TITLE: Runtime command boundary - clean-port plan

* Problem statement
The current runtime command boundary has improved shape with =RuntimeCommand=, =RuntimeCommandKind=, =runtime_command_from_order()=, and =execute_runtime_command()=, but it is still a bridge.

The boundary still:
- adapts mutable legacy dict payloads instead of typed request payloads,
- branches on legacy =ordType= strings for execution,
- calls legacy order helper functions from =kolabi.runtime.kola.orders.orders=.

This keeps old behaviour, but it couples the new core to transitional glue.

* Target boundary
The boundary should become:
- input: typed order intent (=RuntimeCommand= with typed request payload),
- runtime execution: one protocol-based executor that calls exchange ports,
- output: typed ack/result object with a legacy-adapter only at outer edges.

Keep legacy adapters only in one explicit compatibility module.

* What needs doing
** 1. Introduce a typed execution port
Add a protocol in shared core for runtime execution.

- =submit_new_order(request: NewOrderRequest) -> OrderAck=
- =submit_amend_order(request: AmendOrderRequest) -> OrderAck=
- =submit_cancel_order(request: CancelOrderRequest) -> OrderAck=

# > go ahead  

Notes:
- Keep method names literal for typed semantics.
- Do not expose legacy =ordType= mutation semantics through this port.

** 2. Move legacy mapping into one compatibility adapter
Create a runtime compatibility adapter module that:
- accepts the typed request payloads above,
- translates to legacy helper arguments only inside that adapter,
- is the single caller of =place=, =place_stop=, =amend_prices=, =cancel_order=.

Then =execute_runtime_command()= depends on the typed execution port only.
# > I do not care about break functionality. I do not want to bother myself with compatibility layers/adapters

** 3. Remove dict-shape guessing from command building
Split =runtime_command_from_order()= into two layers:
- compatibility parser: legacy order dict -> typed request payload,
- canonical constructor: typed payload -> =RuntimeCommand=.

This isolates TSV and legacy grammar handling from runtime core dispatch.
# > ok

** 4. Replace ordType branching with command payload dispatch
In =execute_runtime_command()=:
- dispatch by =RuntimeCommandKind= and typed payload type,
- avoid direct key pops from mutable dicts,
- make unsupported command/payload combinations fail with explicit typed errors.

** 5. Add deterministic contract tests at the new boundary
Add tests that lock the boundary contract:
- parser tests: legacy dicts map to typed requests,
- executor tests: typed requests call the correct execution-port methods,
- compatibility tests: legacy helper adapter parity for place/amend/cancel paths.

Include frozen fixtures for head/tail and amend/cancel cases.
# > leave the test out for now, or propose a test driven approach

** 6. Define a deprecation sequence
Use three milestones:
- M1: typed execution port + compatibility adapter in place,
- M2: Chronos and async Chronos emit only typed payload commands,
- M3: legacy helper calls removed from main executor; retained only in compatibility module.
# >  we can break the behaviour, to go faster and cleaner from the start
This preserves behaviour while shrinking risk at each step.

* Suggested first implementation slice
1) Add the execution port protocol and typed executor dispatcher.
2) Introduce a compatibility adapter that wraps existing legacy helper calls unchanged.
3) Switch =execute_runtime_command()= to call the adapter via the protocol.
4) Add tests proving no behaviour drift for current order types.

This yields a clean boundary without forcing a full exchange rewrite in one go.
